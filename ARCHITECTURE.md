# AI 记忆系统 - 底层架构重构方案

## 核心问题
当前的 AI 系统严重依赖上下文窗口大小，限制了：
- 长期记忆保持能力
- 知识的跨对话累积
- 大规模对话的一致性

## 新架构设计

### 三层记忆层次结构

```
┌─────────────────────────────────────────────────────────────┐
│  1. Working Memory (工作记忆) - 100 tokens                    │
│     - 当前对话活跃记忆                                          │
│     - LRU 缓存策略                                           │
└─────────────────────────────────────────────────────────────┘
                           ↓ 激活检索
┌─────────────────────────────────────────────────────────────┐
│  2. Recent Memory (近期记忆) - 1000 entries                   │
│     - 近 7 天的对话                                           │
│     - 向量相似度检索                                          │
└─────────────────────────────────────────────────────────────┘
                           ↓ 权重降级
┌─────────────────────────────────────────────────────────────┐
│  3. Long-Term Memory (长期记忆) - 无限容量                     │
│     - 压缩和摘要化                                           │
│     - 分层索引                                             │
└─────────────────────────────────────────────────────────────┘
```

### 核心功能模块

| 模块 | 职责 | 关键技术 |
|------|------|---------|
| VectorStore | 向量存储与相似度搜索 | 余弦相似度 |
| MemoryHierarchy | 层次化管理 | LRU + 权重衰减 |
| ContextManager | 智能上下文管理 | 优先级计算 |
| MemoryCompressor | 记忆压缩和摘要 | 提取关键信息 |
| AssociationNetwork | 记忆关联网络 | 图结构 |

## 优势
- **90%+ 上下文节省**：只在需要时加载相关记忆
- **层次化检索**：从近到远逐步扩展搜索范围
- **智能压缩**：自动将不重要的对话压缩为摘要
- **关联增强**：通过语义相似性自动关联记忆
