# new-agent-memory

## 概述

这是一个探索**类人记忆系统**的实验性项目，通过对人类记忆机制的模仿，构建具有动态权重、分层存储、靶向检索能力的记忆系统。

> "AI应该像人类一样慢慢积累、越用越聪明，而不是像大模型那样'瞬间成熟'。"

## 核心理念

- **越用越聪明** — 系统从使用中自适应调整，而不是一蹴而就
- **遗忘是 feature 不是 bug** — 长期不用的记忆权重自然衰减，是进化结果
- **知识有边界、有成长过程** — 像树苗慢慢长出年轮，不应该像压缩包瞬间解压
- **分层存储** — 不是所有记忆都一样重要，分层管理才能高效

## 架构设计

```
┌─────────────────────────────────────────────────────────┐
│                     检索入口                            │
│   自然语言查询 / 照片信息 / 外部数据 → 信息锚点          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│                    核心记忆层                            │
│   • 高权重记忆常驻                                       │
│   • 主动检索命中率高                                     │
│   • 7因子动态权重计算                                    │
│   • Hebbian 关联网络                                     │
└─────────────────────────────────────────────────────────┘
                            ↓ 权重低于阈值
┌─────────────────────────────────────────────────────────┐
│                    伪遗忘层                              │
│   • 极低权重记忆归档                                     │
│   • 不主动检索，需要信息锚点唤醒                          │
│   • 可被清理，不影响系统运行                             │
│   • "记不起来但看到照片会想起来"                          │
└─────────────────────────────────────────────────────────┘
```

## 动态权重系统

每个记忆碎片的权重由以下因子动态计算：

| 因子 | 作用 | 机制 |
|------|------|------|
| 时间衰减 | 越老权重越低 | 指数衰减，关联多的衰减更慢 |
| 使用频率 | 越常用权重越高 | 对数增长，有天花板 |
| 近因效应 | 最近访问的权重更高 | 24小时内有加成 |
| 情绪增强 | 情绪强烈的事件记得更牢 | 正面/负面情绪直接影响 |
| 关联密度 | 关联越多越稳定 | Hebbian 双向增强 |
| 重要性 | 生存/情感价值 | 基础权重系数 |
| 连接价值 | 能触发多少其他记忆 | 辅助权重 |

```
W_final = Σ(coef_i × factor_i)
```

### Hebbian 关联机制

> "一起放电的神经元，连接会加强"

```python
# 同时访问的记忆互相增强
strengthen_association(memory_A, memory_B)

# 长期不一起使用则衰减
weaken_association(memory_A, memory_B)
```

## 伪遗忘层机制

人类很难主动想起"10年前中午吃了什么"，但看到照片就能想起来。

**伪遗忘层模拟这一过程：**

```
用户问："10年前中午吃了什么"
    ↓
[阶段1] 核心层检索 → 没有
    ↓
[阶段2] 信息锚点扫描伪遗忘层
    • 时间锚点：10年前 + 中午
    • 地点锚点：（如果有）
    • 人物锚点：（如果有）
    ↓
[阶段3] 命中碎片 → 唤醒 + 重组
    ↓
[阶段4] 审阅：判断记忆是否合理
    ↓
[阶段5] 输出（或标记为不可信）
```

## 标签系统

每个记忆碎片携带多维标签，标签即检索的"头文件"：

```
标签签名示例：
T:2026-04-29 | TR:10年前 | TC:中午 | L:北京 | P:我,P:朋友A | TOP:food,TOP:travel
```

| 维度 | 标签类型 | 说明 |
|------|---------|------|
| 时间 | time_absolute / time_relative / time_context | 绝对/相对/上下文时间 |
| 空间 | location / location_detail | 地点 |
| 人物 | persons | 涉及的人物 |
| 主题 | topics / keywords | 主题/关键词 |
| 情绪 | emotion_valence / emotion_intensity | 效价/强度 |
| 元信息 | importance / connection_value | 重要性/连接价值 |

## 检索流程

```
输入（问句/照片）→ QueryContext → 标签解析
    ↓
核心层检索（高权重记忆）
    ↓ 命中
Hebbian 关联扩展（拉取相关记忆）
    ↓
碎片组装 + 审阅
    ↓
输出
    ↓ 未命中
伪遗忘层唤醒（信息锚点匹配）
    ↓ 唤醒成功
碎片组装 + 审阅
    ↓
输出（或标记为不可信）
```

## 审阅机制

重建后的记忆经过审阅，判断是否合理：

| 结果 | 含义 | 处理 |
|------|------|------|
| approved | 合理，直接输出 | — |
| modified | 需要修正后输出 | 尝试修正或降权 |
| questionable | 存疑，谨慎对待 | 降低置信度 |
| rejected | 不合理，标记 | 不输出，标记为伪记忆 |

审阅检查：
- 时间线一致性
- 情绪方向一致性
- 碎片数量与置信度关系
- 历史审阅状态

## 使用示例

```python
from main import HumanLikeMemorySystem

system = HumanLikeMemorySystem()

# 添加记忆
system.add_memory(
    content="今天中午和客户在北京餐厅吃了烤鸭",
    time_absolute="2026-04-29",
    time_context="中午",
    location="北京",
    persons=["客户"],
    topics={"food", "business"},
    emotion_valence=0.3,
    importance=0.7,
)

# 检索记忆
result = system.retrieve("10年前中午吃了什么")

print(result.assembled_content)
print(f"检索路径: {result.retrieval_path}")  # core / forgotten
print(f"置信度: {result.confidence}")

# 通过照片检索
photo_info = {
    "timestamp": 1459200000,
    "location": "北京",
    "faces": ["我", "客户"],
    "labels": ["烤鸭", "餐厅"],
}
result = system.retrieve_by_photo(photo_info)

# 用户反馈
system.feedback("10年前中午吃了什么", accepted=True)  # 被接受
# 或者
system.feedback("10年前中午吃了什么", accepted=False, 
                corrected_content="不是烤鸭，是火锅")
```

## 设计原则

1. **简化问题，抓主要矛盾** — 先解决核心层和伪遗忘层的分离
2. **约束降级处理** — 次要约束作为惩罚项，不必全部堆砌
3. **不追求完美解** — 只求趋于完美的近似解
4. **观察内在规律** — 利用规律简化模型，而非照搬所有机制
5. **越用越聪明** — 系统从反馈中自适应调整
