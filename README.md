# new-agent-memory

## 概述

这是一个探索**类人记忆系统**的实验性项目，通过对人类记忆机制的模仿，构建具有动态权重、分层存储、靶向检索能力的记忆系统。

> "AI应该像人类一样慢慢积累、越用越聪明，而不是像大模型那样'瞬间成熟'。"
> — 项目的核心哲学

## 核心理念

- **越用越聪明** — 系统从使用中自适应调整，而不是一蹴而就
- **遗忘是 feature 不是 bug** — 长期不用的记忆权重自然衰减，是进化结果
- **知识有边界、有成长过程** — 像树苗慢慢长出年轮，不应该像压缩包瞬间解压
- **分层存储** — 不是所有记忆都一样重要，分层管理才能高效
- **不确定性是本质** — 情绪系数带随机分布，极端情绪的系数在不同场景下会有波动
- **不预设人格** — 系统从行为反馈中学习用户是什么人，而非预设偏好

## 自适应权重系统

### 核心理念

权重不是线性累加，而是：

```
有效权重 = 基准关注度 × 情绪修正系数 × 互动量时效衰减
```

- **基准关注度**：用户对这件事的长期重视程度，会随使用情况缓慢漂移
- **情绪修正系数**：上次强化时的情绪采样，极端情绪带来更大的权重增量
- **互动量时效衰减**：基于"我们互动了多少次"而非真实时间

### 情绪系数的概率分布

情绪不是固定系数，而是概率分布采样——这让系统充满"不确定的刺激感"：

| 情绪标签 | 基础均值 | 波动范围(spread) | 说明 |
|---------|---------|-----------------|------|
| 平静 | 0.7 | 0.2 | 低且稳定 |
| 开心 | 1.0 | 0.3 | 正常 |
| 好奇 | 1.1 | 0.3 | |
| 困惑 | 1.3 | 0.4 | |
| 期待 | 1.4 | 0.4 | |
| 难过 | 1.6 | 0.5 | |
| 焦虑 | 1.8 | 0.6 | 高但波动大 |
| 兴奋 | 1.8 | 0.5 | |
| 愤怒 | 2.0 | 0.7 | 高且不稳定 |
| 恐惧 | 2.2 | 0.7 | |
| 崩溃 | 2.5 | 0.8 | 最高且最不稳定 |

> 崩溃和愤怒的spread是0.7~0.8，意味着同样说"崩溃"，不同场景下情绪系数可能在1.7~3.0之间波动。这种不确定性是刻意的。

### 互动量驱动的衰减

| 记忆类型 | 半衰期（互动次数） | 理由 |
|---------|-----------------|------|
| STORY（用户故事） | 500 | 故事是identity的一部分，衰减最慢 |
| IDEA（想法/观点） | 400 | 想法会演化，但不应该被快速遗忘 |
| PREFERENCE（偏好） | 300 | 偏好可能随时间改变 |
| FACT（事实） | 200 | 具体事实衰减较快 |
| INTERACTION（我们之间的事） | 150 | 衰减最快 |

半衰期基于"互动量"而非真实时间——用户连续使用时记忆衰减得很慢，沉默后衰减加速。

### 基准关注度漂移机制

基准关注度会随时间缓慢向当前值回归（漂移率5%）——很久没被强化的记忆，其基准会缓慢下降，但不会突然消失。一旦用户重新提起，权重会立刻重建。

## 分层存储

```
┌─────────────────────────────────────────────────────────┐
│                     核心记忆层                            │
│   有效权重 >= 0.5                                       │
│   高权重记忆常驻，主动检索命中率高                         │
└─────────────────────────────────────────────────────────┘
                            ↓ 权重跌破 0.5
┌─────────────────────────────────────────────────────────┐
│                     伪遗忘层                              │
│   有效权重 < 0.5 但 > 0.05                              │
│   不占核心存储，需要锚点触发唤醒                          │
│   "记不起来但看到照片会想起来"                            │
└─────────────────────────────────────────────────────────┘
                            ↓ 权重 < 0.05
┌─────────────────────────────────────────────────────────┐
│                      彻底删除                            │
└─────────────────────────────────────────────────────────┘
```

## 人格适应层

**不预设人格，从行为反馈中学习。**

系统默认主动提及旧记忆（enabled=True），用户通过反馈告诉系统他是否喜欢这个行为：

| 反馈信号 | 分值 | 说明 |
|---------|------|------|
| 用户继续聊这个话题 | +0.2 | 正向 |
| 用户说"你居然记得这个！" | +0.3 | 强烈正向 |
| 用户转移话题 | -0.15 | 负向 |
| 用户说"别老提这个" | -0.4 | 强烈负向 |
| 用户主动提起旧事 | +0.2 | 正向 |
| 重建内容被纠正 | +0.05 | 愿意纠正说明在关注 |

**学习阈值**：
- 收集满10个信号后，根据兴趣度决定开启或关闭该行为
- 之后每20个信号重新评估一次
- 兴趣度 > 0.6 → 开启，< 0.3 → 关闭

## 审阅机制

重建后的记忆经过审阅，判断是否合理：

| 结果 | 含义 | 处理 |
|------|------|------|
| approved | 合理，直接输出 | — |
| modified | 需要修正后输出 | 尝试修正或降权 |
| questionable | 存疑，谨慎对待 | 降低置信度 |
| rejected | 不合理，标记 | 不输出，标记为伪记忆 |

审阅检查：时间线一致性、情绪方向一致性、碎片数量与置信度关系。

## 设计原则

1. **简化问题，抓主要矛盾** — 先解决核心层和伪遗忘层的分离
2. **约束降级处理** — 次要约束作为惩罚项，不必全部堆砌
3. **不追求完美解** — 只求趋于完美的近似解
4. **观察内在规律** — 利用规律简化模型，而非照搬所有机制
5. **越用越聪明** — 系统从反馈中自适应调整
6. **拥抱不确定性** — 情绪系数不是精确值，而是概率分布
7. **不预设人格** — 从行为反馈中学习用户是什么人
